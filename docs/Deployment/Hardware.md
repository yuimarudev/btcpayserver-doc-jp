# ハードウェアデプロイ

ハードウェアデプロイは、自分のインフラを完全にコントロールしたい場合に推奨されます。

個人利用や、インフラを絶対的に管理したい人にとって非常に有用です。

ハードウェアデプロイの主な欠点は、ハードウェア投資が必要で学習コストが高いことです。LunaNode のような VPS と比べると、インターネット障害やハード故障の影響を受けやすいため、サーバー可用性が下がる可能性もあります。

ハードウェアデプロイが必要かまだ迷っている場合は、[図](./README.md)をご覧ください。

どのハードウェアデプロイも基本は似ていますが、このドキュメントでは3種類のハードウェア構成で手順を段階的に説明しています。

- [Raspberry Pi 4](./RaspberryPi4.md)
- [Hack0](./Hack0.md)（Armbian と RockPro64 ベース）
- [LightningInABox](./LightningInABox.md)（Ubuntu と Gigabyte Brix GB-BXBT-1900 ベース）

このドキュメントは好みのカスタムハードウェアにも容易に応用できます。arm32、arm64、amd64 をサポートしています。

これらのソリューションはすべて docker デプロイシステムに基づいています。
